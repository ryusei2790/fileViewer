# Phase 0：問題定義

## 0-1. ペインの抽出

- **開発者・学習者が**、ローカルファイル（PDF、HTML、画像など）をブラウザで開くと、他のタブやWebサイトが目に入り**集中力が途切れてしまう**
- **開発者・学習者が**、Chrome等の汎用ブラウザを使っていると、**ついWebクローリング（ネットサーフィン）を始めてしまい**、本来の作業に戻れなくなる
- **開発者・学習者が**、ローカルファイルを閲覧するだけなのに、**重いブラウザを起動する必要があり**、リソースを無駄に消費している

## 0-2. 選定されたアプローチ

**Tauriベースの軽量ビューア**を採用。

| 選定理由 |
|----------|
| メモリ使用量がElectronの約1/10。「軽量」要件に最適 |
| フロントエンドはWeb技術（Svelte）で構築可能 |
| Rust学習の機会にもなる |
| ネイティブに近いパフォーマンス |

## 0-3. ターゲットの明確化

```
ターゲットユーザー : 開発者・学習者（自分自身）
ターゲットペイン   : ブラウザでローカルファイルを開くと、Webに気が散って集中できない
解決アプローチ     : ローカルファイル専用の軽量ビューアをTauriで構築し、Web接続を完全に排除する
```

## 0-4. ツールの概念定義

このツールは、ローカルファイル（PDF, HTML, Markdown, 画像, テキスト, CSV, JSON, YAML）を1つのウィンドウで閲覧できる**軽量デスクトップアプリ**である。インターネット接続機能を持たず、ファイル選択ダイアログまたはドラッグ&ドロップでファイルを開く。これにより「ブラウザを開く → 脱線する」というペインを根本から取り除く。

### スケール見込み

個人ツールとしてスタートするが、プラグイン的にファイルタイプ対応を追加できる設計にすれば、他ユーザーにも配布可能。拡張性を考慮した設計にする。
